# 数据隐私与合规声明

> BTC-Pulse 项目隐私合规设计文档

---

## 1. 数据分类与来源审查

| 数据类型 | 来源 | 是否涉及个人信息 | 合规状态 |
|---------|------|:---------------:|---------|
| BTC 价格/成交量 | Bitcoinity (公开CSV)、交易所公开API | ❌ | ✅ 公开市场数据 |
| 恐惧贪婪指数 | Alternative.me 公开API | ❌ | ✅ 聚合统计指标 |
| 资金费率/未平仓量 | Binance/OKX/Bybit 公开API | ❌ | ✅ 匿名聚合衍生品数据 |
| 新闻标题 | CoinDesk/CoinTelegraph RSS | ❌ | ✅ 公开发布内容 |
| LLM 调用 | DeepSeek/Ollama/OpenAI API | ⚠️ 传输内容 | 见下方说明 |

**结论：BTC-Pulse 全部数据来源均为公开市场数据和聚合统计指标，不涉及任何用户个人信息（PII）、交易账户数据或钱包地址。**

---

## 2. 隐私风险识别与处理

### 2.1 LLM API 调用风险

**风险描述：** `models/llm_sentiment.py` 将新闻标题发送至外部 LLM API 进行情感分析，存在数据传输至第三方风险。

**处理措施：**
- 仅传输**公开新闻标题**（非用户数据），属于已公开信息再处理
- 支持**本地 Ollama 部署**（`ai_provider: "ollama"`），实现零数据外传
- 不传输任何用户账户、密码、API Key 等敏感信息至 LLM
- System prompt 中不包含任何个人可识别信息

### 2.2 API Key 存储风险

**风险描述：** `config.yaml` 存储交易所 API Key 和 LLM API Key。

**处理措施：**
- `config.yaml` 已加入 `.gitignore`，不会被提交到代码仓库
- 仅提供 `config.yaml.example` 模板（不含真实密钥）
- 代码中通过 `utils/config.py` 的 `get_api_key()` 统一管理，不硬编码密钥
- 推荐用户使用环境变量覆盖配置文件中的敏感字段

### 2.3 本地数据库安全

**风险描述：** SQLite 数据库存储历史市场数据。

**处理措施：**
- 数据库文件 `data/btc_pulse.db` 已加入 `.gitignore`
- 数据库仅存储公开市场数据，不含任何个人信息
- SQLite 使用 WAL 模式，`init_db.py` 实现**表名白名单校验**防止 SQL 注入

### 2.4 日志隐私

**处理措施：**
- Loguru 日志不记录 API Key 等敏感信息
- 日志仅记录操作状态和市场数据统计信息

## 3. 合规设计原则

| 原则 | 实践 |
|------|------|
| **数据最小化** | 仅采集分析所需的市场公开数据，不采集用户行为或个人信息 |
| **本地优先** | 所有数据存储在本地 SQLite，不上传至任何云服务 |
| **零PII设计** | 系统设计上不涉及个人可识别信息的采集、存储或处理 |
| **密钥隔离** | 配置文件与代码分离，敏感配置不进入版本控制 |
| **可审计** | 所有数据来源、处理流程在代码中可追溯 |
